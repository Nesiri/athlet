Complete CSS Selectors Cheat Sheet

This document provides a comprehensive overview of all CSS selectors, organized into sections for easy reference.

1. Basic Selectors

Universal Selector (*): Selects all elements on the page.

Type Selector (e.g., div, p): Selects all instances of a specific element type.

Class Selector (.class-name): Selects elements with a specific class.

ID Selector (#id-name): Selects an element with a unique ID.

2. Attribute Selectors

[attribute]: Selects elements that have a specific attribute.

[attribute=value]: Selects elements with an attribute matching a specific value.

[attribute~=value]: Selects elements with an attribute containing a specific word.

[attribute|=value]: Selects elements with an attribute value that starts with "value".

[attribute^=value]: Selects elements where an attribute starts with a specific value.

[attribute$=value]: Selects elements where an attribute ends with a specific value.

[attribute*=value]: Selects elements where an attribute contains a specific substring.

3. Combinator Selectors

Descendant Selector (A B): Selects elements inside another element, regardless of depth.

Child Selector (A > B): Selects direct child elements only.

Adjacent Sibling (A + B): Selects the immediate sibling following a specified element.

General Sibling (A ~ B): Selects all siblings that follow a specific element.

4. Pseudo-Classes

:first-child: Selects the first child element within a parent.

:last-child: Selects the last child element within a parent.

:nth-child(n): Selects elements based on their position within a parent.

:nth-last-child(n): Selects elements counting from the end within a parent.

:nth-of-type(n): Selects elements of a specific type in their position order.

:nth-last-of-type(n): Selects elements of a specific type counting from the end.

:only-child: Selects elements that are the only child within a parent.

:only-of-type: Selects elements that are the only instance of their type within a parent.

:empty: Selects elements with no children, including text nodes.

:root: Selects the root element, typically .

:hover: Applies styles when an element is hovered.

:focus: Applies styles when an element is focused.

:active: Targets elements in an active state, such as clicked buttons.

:visited: Applies styles to visited links.

:checked: Selects checked form elements, such as checkboxes.

5. Pseudo-Elements

::before: Inserts content before an element's actual content.

::after: Inserts content after an element's actual content.

::first-line: Styles the first line of an element's content.

::first-letter: Styles the first letter of an element's content.

::placeholder: Styles placeholder text in input elements.

::selection: Applies styles to selected text.

::marker: Styles the marker of list items, such as bullets or numbers.

6. Advanced Selectors

:not(selector): Negates a selector, selecting all elements that do not match the specified selector.

:is(selector): Matches any element within a list of selectors.

:where(selector): Matches elements within a list without affecting specificity.

:has(selector): Selects elements containing specific descendants (experimental).

:focus-within: Applies styles to an element if any of its descendants are focused.

:focus-visible: Targets focused elements that should visually indicate focus.

CSS Selectors Cheat Sheet

* Universal Selector

The universal selector applies to all elements on the page.
Difference: Targets all elements, unlike more specific selectors.
Similarity: Like type selectors, it applies to multiple elements, but not specific to any type.
Use Case: Apply global styles (e.g., box-sizing) across all elements.
Preferred: When a broad application of styles is needed.

element Type Selector (e.g., div, p)

Selects all instances of a specified HTML element.
Difference: Targets only specific elements, unlike the universal selector.
Similarity: Applies to all elements of the specified type, like the universal selector applies to all elements.
Use Case: Useful for setting consistent styles for specific HTML elements.
Preferred: When styling a specific element type across the page.

.class-name Class Selector

Targets all elements with a specific class attribute.
Difference: More specific than the universal or type selectors and can apply to multiple elements.
Similarity: Like the ID selector, used for targeted styling but can apply to multiple elements.
Use Case: Commonly used for grouping styles for multiple elements.
Preferred: For reusable styles across various elements.

#id-name ID Selector

Selects a unique element with a specific ID.
Difference: IDs are unique; classes can apply to multiple elements.
Similarity: Both are used for targeted styling, but ID has higher specificity.
Use Case: Used for unique styling of one specific element.
Preferred: When an element requires a unique style, but avoid overuse as it limits reusability.

[attribute] Attribute Selector

Targets elements containing a specific attribute.
Difference: Targets based on attribute presence rather than type or class.
Similarity: Similar to the class selector, but based on attributes rather than classes.
Use Case: Styling elements with particular attributes, like input[type="text"].
Preferred: When styling elements dynamically based on their attributes.

:nth-child(n) vs. :nth-of-type(n)

:nth-child(n) selects elements based on position within their parent, regardless of type. :nth-of-type(n) selects elements based on position within their parent of a specific type.
Difference: :nth-child(n) matches any child element by position, while :nth-of-type(n) matches only specific types at a given position.
Similarity: Both are pseudo-classes that use position for styling.
Use Case: :nth-child is used when any element at a position needs styling; :nth-of-type is used when only specific types at positions need styling.
Preferred: Use :nth-of-type when you need to style specific elements of a type within a set.

Descendant (A B) vs. Child (A > B)

Descendant Selector (A B) targets elements nested within specified parents at any depth. Child Selector (A > B) targets only direct children of a specified parent.
Difference: Descendant allows any depth, while Child targets only immediate children.
Similarity: Both are combinators for parent-child relationships.
Use Case: Use Descendant when styling all nested elements; Child for direct children.
Preferred: Child is better for specific parent-child targeting to avoid unintended styling.

:not(selector) Pseudo-Class Selector

Selects elements that do not match the specified selector.
Difference: Inverse of all other selectors, selects based on exclusion.
Similarity: Can work with any selector type to exclude matches.
Use Case: Useful for styling elements that do not match specific criteria.
Preferred: Use for precise styling control when needing to exclude specific elements.

CSS Cheat Sheet

This guide provides descriptions and usage examples for common CSS properties.

Text and Font Properties

color
Sets the color of the text.
color: blue;
font-size
Specifies the size of the font.
font-size: 16px;
font-family
Sets the font type for text.
font-family: Arial, sans-serif;
font-weight
Adjusts the boldness of text.
font-weight: bold;
text-align
Specifies the horizontal alignment of text.
text-align: center;
text-decoration
Defines decorations for text like underline or line-through.
text-decoration: underline;
text-transform
Modifies text capitalization.
text-transform: uppercase;

Background Properties

background-color
Sets the background color.
background-color: #f0f0f0;
background-image
Sets a background image.
background-image: url('image.jpg');
background-repeat
Defines if/how a background image repeats.
background-repeat: no-repeat;
background-position
Sets the starting position of a background image.
background-position: center center;
background-size
Specifies the size of a background image.
background-size: cover;

Box Model Properties

width
Defines the width of an element.
width: 200px;
height
Defines the height of an element.
height: 100px;
padding
Sets the padding inside an element's border.
padding: 10px;
margin
Sets the space outside an element's border.
margin: 20px;
border
Sets the border of an element.
border: 1px solid #000;
border-radius
Defines rounded corners for elements.
border-radius: 5px;
box-shadow
Adds shadow to an element's box.
box-shadow: 2px 2px 5px rgba(0, 0, 0, 0.5);

Display and Positioning

display
Specifies how an element is displayed.
display: inline-block;
position
Defines the positioning type (static, relative, absolute, fixed, sticky).
position: relative;
top, right, bottom, left
Offsets an element from the respective side when using position (except static).
top: 10px; left: 5px;
float
Moves elements to the left or right, allowing other content to wrap around.
float: left;
clear
Stops elements from wrapping around floated elements.
clear: both;
z-index
Sets the stack order of positioned elements.
z-index: 10;

Flexbox Properties

display: flex
Activates flexbox layout on a container.
display: flex;
justify-content
Aligns items horizontally within a flex container.
justify-content: center;
align-items
Aligns items vertically within a flex container.
align-items: center;
flex-direction
Sets the main axis direction.
flex-direction: row;
flex-wrap
Controls if items wrap to new lines.
flex-wrap: wrap;
align-self
Overrides align-items for individual items.
align-self: flex-end;

Grid Properties

display: grid
Activates grid layout on a container.
display: grid;
grid-template-columns
Defines the number and width of columns in a grid.
grid-template-columns: 1fr 2fr;
grid-template-rows
Defines the number and height of rows in a grid.
grid-template-rows: 100px auto;
gap
Sets space between grid items.
gap: 10px;
grid-column
Specifies the column position of a grid item.
grid-column: 1 / 3;
grid-row
Specifies the row position of a grid item.
grid-row: 1 / 2;
